# AI Orchestrator Wiki Schema

## Ownership

- `raw/` is immutable source-of-truth material. Agents never edit ingested raw
  files. A corrected source is added as a new, versioned file.
- `wiki/pages/` is the compiled knowledge layer. Agents may create and update
  Markdown pages there.
- `state/manifest.json` records source hashes and ingest provenance.
- `wiki/index.md` is regenerated by tooling.
- `wiki/log.md` is append-only.

## Page conventions

- Keep one clear topic per page.
- Preserve source paths and SHA-256 citations.
- Use `[[wikilinks]]` for durable relationships.
- Mark mechanically generated pages as `status: draft`.
- An agent may promote a reviewed page to `status: curated`.
- Record contradictions instead of silently choosing one claim.

## Operations

1. Ingest: read a raw source once, create a compact draft page, update the
   manifest/index, and append the log.
2. Query: search the wiki first and read only returned pages. Read raw files only
   when verifying a citation or resolving a documented gap.
3. Lint: check immutable hashes, source/page coverage, index coverage, and
   unresolved wikilinks.
4. Curate: agents enrich draft pages, add synthesis/entity/concept pages, and
   preserve citations.
