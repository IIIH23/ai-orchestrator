# Partner Profile Template

Purpose: Capture collaboration preferences, role boundaries, and operating principles for project partners.

Fields:

- Name: <Partner or organization name>
- Contact: <preferred contact method — email, Matrix, Telegram, etc.>
- Role(s): <e.g. data provider, analyst, reviewer, maintainer>
- Responsibilities: 
  - <Primary responsibilities>
  - <Secondary responsibilities>
- Availability / Timezone: <e.g. UTC+2, office hours>
- Communication preferences:
  - Frequency: <daily/weekly/as-needed>
  - Channels: <email/Matrix/Slack/Telegram>
  - Escalation path: <who to contact when issues arise>
- Access constraints: <what resources they should not be given — e.g., no deploy keys>
- Security & privacy notes: <PII handling, data retention limits>
- Onboarding checklist:
  - Introductions & expectations
  - Access to required data
  - Read ROADMAP.md and AGENTS.md
- Offboarding checklist:
  - Revoke access
  - Archive deliverables

Usage notes:
- Fill this template per partner and store under docs/partners/<partner-name>.md
- Keep profiles small and review them when responsibilities change.
